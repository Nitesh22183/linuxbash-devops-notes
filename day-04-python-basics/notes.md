# Day 4 Notes - Python Basics for DevOps

## Why Python is useful in DevOps
Python is used in DevOps for automation, API calls, file handling, health checks, cloud scripting, and tool integration.

## How Python differs from Bash
Bash is good for shell commands and Linux tasks.
Python is better for structured logic, API calls, JSON handling, and larger automation scripts.

## What is a Python script?
A Python script is a file ending in `.py` that contains Python code and can be executed using `python3 filename.py`.

## Variables
Variables store values.

## Basic data types
- string
- integer
- list
- dictionary
- boolean

## Function
A function is a reusable block of code.

## File handling
Python can open and read files, which is useful for logs, config files, and automation.

## Indentation
Python uses indentation to define code blocks instead of curly braces.


## try/except
`try/except` is used to catch errors and prevent script failure.

This is useful in DevOps scripts when files, APIs, or services may fail.

## requests library
The `requests` library is used to call URLs and APIs in Python.

It is useful for:
- health checks
- API automation
- cloud integrations
